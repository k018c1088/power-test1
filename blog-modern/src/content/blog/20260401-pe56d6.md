## バージョン管理とは？

バージョン管理とは、ファイルの変更履歴を記録・管理するシステムです。

例えば、レポートを書く時に「最終版.docx」「最終版_修正.docx」「最終版_修正2.docx」のようにファイル名を変えて保存した経験はありませんか？これも一種のバージョン管理ですが、プログラミングではもっと効率的な方法を使います。

### バージョン管理の利点

- **変更履歴の追跡**: いつ、誰が、何を変更したかが分かる
- **巻き戻し機能**: 問題があった時に前の状態に戻せる
- **チーム開発**: 複数人で同じプロジェクトを安全に編集できる

## GitとGitHubの違い

- **Git**: バージョン管理システム（ツール）
- **GitHub**: Gitを使ったプロジェクトをオンラインで管理するサービス

GitHubは「Gitのクラウド版」と考えると分かりやすいですね。

## GitHubの基本的な使い方

### 1. アカウント作成とリポジトリ作成

1. [GitHub](https://github.com)でアカウントを作成
2. 「New repository」をクリック
3. リポジトリ名を入力（例：my-first-project）
4. 「Create repository」をクリック

### 2. 基本的なGitコマンド

```bash
# リポジトリをローカルにコピー
git clone https://github.com/username/my-first-project.git

# ファイルの変更を追加
git add .

# 変更をコミット（記録）
git commit -m "初回コミット"

# GitHubに変更をアップロード
git push origin main
```

### 3. 実際の作業フロー

1. **ファイルを編集**
2. **変更を確認**: `git status`
3. **変更を追加**: `git add ファイル名`
4. **コミット**: `git commit -m "変更内容の説明"`
5. **プッシュ**: `git push`

## 新卒エンジニアが覚えるべき重要な概念

### ブランチ（Branch）

ブランチは「並行した開発ライン」です。メインの開発を邪魔せずに新機能を開発できます。

```bash
# 新しいブランチを作成・移動
git checkout -b feature/new-function

# ブランチ一覧を確認
git branch
```

### プルリクエスト（Pull Request）

自分の変更をメインブランチに統合する前に、チームメンバーにレビューしてもらう仕組みです。

## よくある初心者の質問

**Q: コミットメッセージはどう書けば良い？**
A: 「何をしたか」を簡潔に書きましょう。
- 良い例：「ログイン機能を追加」「バグを修正」
- 悪い例：「更新」「変更」

**Q: いつコミットすればいい？**
A: 意味のある単位で区切りましょう。1つの機能や修正が完了したタイミングがおすすめです。
